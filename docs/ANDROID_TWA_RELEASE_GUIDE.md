## Digital Asset Links requirement

A Play Store TWA requires a valid assetlinks file hosted at:

```text
https://lexora.vercel.app/.well-known/assetlinks.json
```

Do not publish a fake certificate fingerprint.

The final file must contain the real SHA-256 certificate fingerprint generated by the Android signing key used for the Play Console build.

Template:

```json
[
  {
    "relation": ["delegate_permission/common.handle_all_urls"],
    "target": {
      "namespace": "android_app",
      "package_name": "com.foxboxstudio.lexora",
      "sha256_cert_fingerprints": [
        "REPLACE_WITH_REAL_PLAY_SIGNING_SHA256_FINGERPRINT"
      ]
    }
  }
]
```

## How to get SHA-256 fingerprint

For a local keystore:

```bash
keytool -list -v -keystore path/to/your-release-key.jks -alias your-key-alias
```

Use the SHA256 value from the certificate fingerprints section.

For Play App Signing:

```text
Play Console -> App integrity -> App signing key certificate -> SHA-256 certificate fingerprint
```

## Release checklist

```text
1. Vercel production URL opens on Android Chrome.
2. Manifest loads from /manifest.webmanifest.
3. Service worker loads from /service-worker.js.
4. App icon loads from /icon.svg.
5. assetlinks.json is hosted with the real Play signing SHA-256 fingerprint.
6. Android package name is com.foxboxstudio.lexora.
7. Internal testing AAB installs correctly.
8. TWA opens without browser address bar.
9. Wheel scroll/switch behavior is correct on a physical Android phone.
10. npm run verify:levels passes before upload.
```
