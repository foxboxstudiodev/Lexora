# Lexora Master Plan

Status: ACTIVE.

This is the new main planning document for Lexora.

## 1. Platform Scope

Lexora is not Android-only.

Lexora is planned for:

1. Android — Google Play Market.
2. iOS — Apple App Store.
3. Web/PWA — supporting channel.

Main release target:

```text
Android + iOS
Google Play Market + Apple App Store
```

Android and iOS must be planned together as first-class mobile release paths.

Android documents cover Android implementation, Google Play packaging, AAB builds, Android screenshots, Android icons, Android testing, and Google Play requirements.

iOS documents must be added next to Android documents and must cover Apple App Store release, iOS builds, iPhone/iPad screenshots where required, iOS icons, signing, TestFlight, App Store review, App Privacy, and Apple policy requirements.

Web/PWA remains a supporting channel, not the primary commercial target.

## 2. Level Structure

Lexora uses 1000 levels per language.

Each language has exactly 20 blocks of 50 levels:

```text
Block 1: 1-50
Block 2: 51-100
Block 3: 101-150
...
Block 20: 951-1000
```

Every 50-level block repeats the same internal difficulty law:

```text
1-10   easy
11-25  medium
26-40  hard
41-50  very hard / final
```

Each new block starts easier relative to that block, but the global difficulty still rises from Block 1 to Block 20.
