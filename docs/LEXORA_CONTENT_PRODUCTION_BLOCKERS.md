# Lexora Content Production Blockers

Status: ACTIVE.

This document lists content blockers that prevent Lexora from being considered production-ready.

## Current blocker: seed-expanded content packs

The current content packs still rely on small seed packs expanded to 300 levels per language.

This is not acceptable for final release under `LEXORA_MASTER_PLAN.md`.

## Why strict release verification fails

The master plan requires each 20-level block to follow exact main-word counts:

- level 1 in each block: 2 mandatory crossword words;
- level 20 in each block: 17 mandatory crossword words.

Several current language packs contain seed entries with only a few source words. For example, a seed entry with 3 source words cannot produce a valid level that requires 17 mandatory crossword words.

Therefore the project needs real production content before final release verification can pass.

## Required fix

For every language:

1. Create 300 real levels.
2. Each level must match the exact 20-level law.
3. Every required main word must be a real noun.
4. Bonus words must also be real nouns.
5. Levels must not be generated by simple seed rotation.
6. Level signatures must be unique.
7. Crossword layouts must be clean, connected, and readable only left-to-right / top-to-bottom.

## Release rule

A Play Market / App Store release is blocked until all seed-expanded levels are replaced by real production-quality content.
